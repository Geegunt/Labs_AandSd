## Вариант 2

**Тема работы:** сбалансированные деревья. B-дерево.

**Задание:** построить упорядоченное дерево для хранения и поиска элементов с уникальными строковыми ключами и вещественными значениями. Ключ — непустая последовательность латинских букв не более чем из 6 символов. Ключи упорядочиваются лексикографически. Вид дерева определяется по формуле (N%3)+1, где N — номер в группе. При N=4 получаем вариант 2 — B-дерево порядка 2 (n=2).

## 1. Цель работы

Целью лабораторной работы является разработка программы на языке C, которая:

1. реализует B-дерево с минимальной степенью t=2 (2-3-4 дерево) для хранения пар ключ–значение;
2. поддерживает операции добавления, удаления, поиска и печати дерева с автоматической балансировкой;
3. принимает команды из текстового файла в заданном формате;
4. выводит результат каждой команды в текстовый файл;
5. поддерживает интерактивный режим работы через консоль.

## 2. Краткие теоретические сведения

### 2.1. Деревья поиска

Дерево поиска — структура данных, в которой для каждого узла выполняется инвариант: все ключи в левом поддереве меньше ключа узла, все ключи в правом поддереве больше. Это позволяет искать, добавлять и удалять элементы за O(h), где h — высота дерева.

Проблема обычного бинарного дерева поиска: при вставке элементов в отсортированном порядке дерево вырождается в список и h = n. Сбалансированные деревья решают эту проблему, поддерживая h = O(log n) при любом порядке вставок.

### 2.2. B-дерево

B-дерево — обобщённое сбалансированное дерево поиска, в котором каждый узел хранит несколько ключей и имеет несколько дочерних узлов. B-дерево всегда остаётся идеально сбалансированным: все листья находятся на одной глубине.

Параметр t называется минимальной степенью дерева. При t=2 получается частный случай — 2-3-4 дерево.

### 2.3. B-дерево порядка 2 (минимальная степень t=2)

| Параметр | Значение |
|---|---|
| Минимальное число ключей в узле (кроме корня) | t−1 = **1** |
| Максимальное число ключей в узле | 2t−1 = **3** |
| Максимальное число дочерних узлов | 2t = **4** |

Ключи внутри каждого узла хранятся в отсортированном порядке. Если у узла k ключей, у него ровно k+1 дочерних узлов (для внутренних узлов). Поддерево между ключами `keys[i]` и `keys[i+1]` содержит только ключи, которые лексикографически больше `keys[i]` и меньше `keys[i+1]`.

### 2.4. Расщепление узла

Когда узел заполнен (3 ключа) и в него нужно вставить ещё один элемент, выполняется расщепление: средний ключ поднимается к родителю, а узел делится на два узла с одним ключом каждый.

```text
До расщепления:
родитель: [fig]
   ребёнок: [apple | cherry | elder]  ← переполнен

После расщепления:
родитель: [cherry | fig]
   левый: [apple]    правый: [elder]
```

### 2.5. Слияние узлов

Обратная операция. Когда узел остаётся с 0 ключами после удаления, два соседних узла с минимальным числом ключей сливаются в один. Ключ-разделитель из родителя опускается вниз.

```text
До слияния:
родитель: [cherry | fig]
   левый: [apple]    правый: [elder]

После слияния:
родитель: [fig]
   ребёнок: [apple | cherry | elder]
```

### 2.6. Лексикографический порядок

Ключи сравниваются как строки: посимвольно слева направо. `"apple" < "banana"`, потому что `'a' < 'b'`. `"apple" < "apricot"`, потому что на третьей позиции `'p' < 'r'`. Для сравнения используется стандартная функция `strcmp`.

## 4. Описание основных структур данных

### 4.1. Узел дерева

```c
typedef struct BTreeNode {
    int n;
    char keys[MAX_KEYS][KEY_LEN];
    double values[MAX_KEYS];
    struct BTreeNode *children[MAX_CHILDREN];
    int leaf;
} BTreeNode;
```

Поля структуры:

- `n` — текущее количество ключей в узле (от 0 до 3);
- `keys` — массив строковых ключей, отсортированных лексикографически;
- `values` — массив вещественных значений, соответствующих ключам;
- `children` — массив указателей на дочерние узлы (используется только если `leaf == 0`);
- `leaf` — флаг: 1 если узел является листом, 0 если внутренним.

У узла с `n` ключами ровно `n+1` дочерних узлов. Поэтому размер массива `children` равен `MAX_CHILDREN = 4`, что на единицу больше `MAX_KEYS = 3`.

### 4.2. Дерево

```c
typedef struct {
    BTreeNode *root;
} BTree;
```

Обёртка, хранящая указатель на корень. Корень может меняться при расщеплении или слиянии.

### 4.3. Константы

```c
#define T 2            // минимальная степень
#define MAX_KEYS 3     // 2*T - 1
#define MAX_CHILDREN 4 // 2*T
#define KEY_LEN 7      // 6 букв + '\0'
```

## 5. Описание функций

### 5.1. Вспомогательные функции создания и освобождения

**`node_create(leaf)`** — выделяет память под новый узел, обнуляет счётчик ключей и все указатели на детей.

**`node_free(node)`** — рекурсивно освобождает всё поддерево: сначала освобождает всех детей (если узел не лист), затем сам узел.

**`btree_create()`** — создаёт пустое дерево с одним пустым корнем-листом.

**`btree_free(tree)`** — вызывает `node_free` для корня, затем освобождает саму обёртку `BTree`.

### 5.2. Поиск

**`node_search(node, key, idx_out)`** — рекурсивный поиск ключа начиная с узла `node`.

| Шаг | Действие |
|---|---|
| 1 | Идём вправо по ключам узла, пока текущий ключ меньше искомого |
| 2 | Если нашли равный ключ — записываем позицию в `idx_out` и возвращаем узел |
| 3 | Если это лист и ключ не найден — возвращаем NULL |
| 4 | Если это внутренний узел — рекурсивно спускаемся в ребёнка `i` |

**`btree_search(tree, key, value_out)`** — публичная обёртка. Запускает `node_search` с корня. При успехе записывает значение через указатель `value_out` и возвращает 1. При неудаче возвращает 0.

### 5.3. Вставка

**`split_child(parent, i, child)`** — расщепление переполненного узла `child`.

| Шаг | Действие |
|---|---|
| 1 | Создать новый узел `new_node` |
| 2 | Скопировать правую половину ключей из `child` в `new_node` (ключи с индекса T и далее) |
| 3 | Если `child` не лист — перенести правую половину детей в `new_node` |
| 4 | Уменьшить `child->n` до T-1 |
| 5 | Вставить `new_node` в массив детей `parent` на позицию `i+1` |
| 6 | Поднять средний ключ (`child->keys[T-1]`) к родителю на позицию `i` |
| 7 | Увеличить `parent->n` |

**`insert_non_full(node, key, value)`** — вставка в гарантированно незаполненный узел.

- Если узел — лист: сдвигает ключи вправо и вставляет в нужную позицию.
- Если узел внутренний: находит нужного ребёнка, при необходимости расщепляет его превентивно, затем рекурсивно спускается вниз.

**`btree_insert(tree, key, value)`** — публичная вставка.

| Шаг | Действие |
|---|---|
| 1 | Если корень переполнен (3 ключа) — создать новый пустой корень, старый сделать его ребёнком, расщепить |
| 2 | Вызвать `insert_non_full` от (нового) корня |

Дерево растёт вверх: при расщеплении корня высота увеличивается на 1, при этом все листья по-прежнему остаются на одной глубине.

### 5.4. Удаление

Удаление состоит из нескольких вспомогательных операций.

**`find_key_idx(node, key)`** — возвращает индекс первого ключа в узле, который не меньше искомого.

**`remove_from_leaf(node, idx)`** — удаляет ключ из листа: сдвигает все ключи после него влево и уменьшает счётчик.

**`get_predecessor_node(node, idx)`** — возвращает лист, содержащий предшественника ключа `idx`. Идёт в левое поддерево и затем всегда вправо до листа.

**`get_successor_node(node, idx)`** — возвращает лист с преемником. Идёт в правое поддерево и затем всегда влево до листа.

**`merge_children(node, idx)`** — слияние детей `idx` и `idx+1`.

| Шаг | Действие |
|---|---|
| 1 | Опустить ключ `node->keys[idx]` из родителя в конец левого ребёнка |
| 2 | Скопировать все ключи и детей из правого ребёнка в конец левого |
| 3 | Удалить ключ и указатель на правого ребёнка из родителя |
| 4 | Освободить правого ребёнка |

**`borrow_from_prev(node, idx)`** — ротация ключа через родителя от левого соседа к ребёнку `idx`. Левый сосед отдаёт свой максимальный ключ родителю, а тот отдаёт свой ключ ребёнку.

**`borrow_from_next(node, idx)`** — симметрично `borrow_from_prev`. Правый сосед отдаёт минимальный ключ.

**`fill_child(node, idx)`** — пополняет дефицитного ребёнка, у которого 0 ключей.

| Условие | Действие |
|---|---|
| Левый сосед имеет ≥ T ключей | `borrow_from_prev` |
| Правый сосед имеет ≥ T ключей | `borrow_from_next` |
| Оба соседа минимальны | `merge_children` с правым (или левым, если правого нет) |

**`remove_from_internal(node, idx)`** — удаление ключа из внутреннего узла.

| Условие | Действие |
|---|---|
| Левый ребёнок имеет ≥ T ключей | Заменить ключ предшественником, рекурсивно удалить предшественник |
| Правый ребёнок имеет ≥ T ключей | Заменить ключ преемником, рекурсивно удалить преемник |
| Оба ребёнка минимальны | Слить детей, рекурсивно удалить ключ из слитого узла |

**`delete_from_node(node, key)`** — основная рекурсивная функция удаления.

| Шаг | Действие |
|---|---|
| 1 | Найти позицию ключа через `find_key_idx` |
| 2 | Если ключ найден в этом узле — удалить (из листа или внутреннего) |
| 3 | Если это лист и ключа нет — вернуть 0 (ключ отсутствует) |
| 4 | Если не лист — превентивно пополнить нужного ребёнка через `fill_child` |
| 5 | Рекурсивно спуститься в нужного ребёнка |

**`btree_delete(tree, key)`** — публичное удаление. После рекурсии проверяет: если корень опустел, но не является листом — его единственный ребёнок становится новым корнем. Дерево уменьшилось на уровень.

### 5.5. Печать

**`print_node(node, level, out)`** — рекурсивный обход дерева сверху вниз.

- Выводит отступ `level × 4` пробелов.
- Выводит ключи узла в формате `[key1:val1 | key2:val2]`.
- Рекурсивно печатает всех детей с `level + 1`.

**`btree_print(tree, out)`** — публичная обёртка, проверяет пустоту дерева и вызывает `print_node`.

## 6. Формат команд

### Входной файл

Каждая строка — одна команда:

| Формат строки | Операция |

### Выходной файл

Для каждой команды выводится:
```
<команда из входного файла>
<результат выполнения>

```

### Пример 1 — добавление и печать

Входные команды:

## 9. Ограничения программы

1. Ключ — только латинские буквы (a–z, A–Z), длина от 1 до 6 символов.
2. Дубликаты ключей не допускаются — повторная вставка существующего ключа сообщает об ошибке.
3. Значения — вещественные числа типа `double`.
4. Длина строки входного файла — не более 255 символов.

## 10. Вывод

В ходе лабораторной работы реализовано B-дерево с минимальной степенью t=2 на языке C. Дерево хранит пары строковый ключ — вещественное значение с лексикографическим упорядочением ключей и поддерживает четыре операции: вставку с расщеплением, удаление со слиянием и ротацией, поиск и печать.

Программа поддерживает два режима работы: интерактивный (ввод с клавиатуры) и файловый (чтение команд из файла с записью результатов в выходной файл). Проведённое тестирование на наборе из 23 элементов подтвердило корректность всех операций и соблюдение инварианта B-дерева: все листья находятся на одной глубине при любом порядке вставок и удалений.
