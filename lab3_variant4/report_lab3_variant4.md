## 1. Цель работы

Целью лабораторной работы является разработка программы на языке C, которая:

1. считывает арифметическое выражение;
2. преобразует его из инфиксной записи в обратную польскую запись;
3. строит дерево выражения;
4. выполняет упрощение некоторых операций деления;
5. выводит исходное выражение, обратную польскую запись, дерево до преобразования, дерево после преобразования и итоговое выражение.

## 2. Краткие теоретические сведения

### 2.1. Инфиксная запись

Инфиксная запись - это обычная форма записи арифметического выражения, в которой оператор находится между операндами.

Пример:

```text
4 * a / 2
```

Здесь операторы `*` и `/` стоят между значениями `4`, `a` и `2`.

### 2.2. Обратная польская запись

Обратная польская запись, или ОПН, - это форма записи, в которой оператор записывается после своих операндов.

Пример:

```text
4 a * 2 /
```

Такая запись удобна для построения дерева выражения, потому что при чтении оператора уже известны его операнды.

Например, для фрагмента:

```text
4 a *
```

оператор `*` применяется к двум предыдущим операндам: `4` и `a`.

### 2.3. Стек

Стек - это структура данных, работающая по принципу LIFO: last in, first out. Это означает: последний добавленный элемент извлекается первым.

В программе стек используется дважды:

1. при переводе выражения в ОПН - для хранения операторов;
2. при построении дерева выражения - для хранения временных узлов дерева.

### 2.4. Дерево выражения

Дерево выражения - это структура, в которой:

- листья дерева являются числами или переменными;
- внутренние узлы являются операторами;
- левое и правое поддерево оператора являются его операндами.

Для выражения:

```text
4 * a / 2
```

дерево имеет вид:

```text
/
    *
        4
        a
    2
```

Корнем дерева является операция деления `/`. Слева от нее находится произведение `4 * a`, справа - число `2`.

## 4. Описание основных структур данных

### 4.1. Узел дерева

Узел дерева описан структурой `Node`:

```c
typedef struct Node {
    char *value;
    struct Node *left;
    struct Node *right;
    bool is_unary;
    bool is_number;
    int num_value;
} Node;
```

Поля структуры:

- `value` - строковое значение узла, например `"+"`, `"*"`, `"4"` или `"a"`;
- `left` - указатель на левое поддерево;
- `right` - указатель на правое поддерево;
- `is_unary` - признак унарного оператора;
- `is_number` - признак того, что узел хранит число;
- `num_value` - числовое значение, если узел является числом.

### 4.2. Стек операторов

Стек операторов описан структурой `OpStack`:

```c
typedef struct {
    char **data;
    int top;
    int capacity;
} OpStack;
```

Поля структуры:

- `data` - динамический массив строк;
- `top` - индекс верхнего элемента стека;
- `capacity` - текущая вместимость массива.

Если стек переполняется, его размер увеличивается с помощью `realloc`.

## 5. Алгоритм работы программы

### 5.1. Чтение выражения

В `main.c` выражение считывается из стандартного ввода. Для чтения используется динамический буфер. Если введенная строка становится больше текущего размера буфера, память увеличивается с помощью `realloc`.

Это позволяет читать выражения произвольной длины, пока хватает оперативной памяти.

### 5.2. Перевод выражения в ОПН

Функция `infix_to_postfix` переводит выражение в обратную польскую запись.

Алгоритм работы:

1. Программа идет по строке слева направо.
2. Если встречено число или переменная, оно сразу добавляется в выходной массив токенов.
3. Если встречена открывающая скобка `(`, она помещается в стек.
4. Если встречена закрывающая скобка `)`, операторы извлекаются из стека до ближайшей открывающей скобки.
5. Если встречен оператор, программа сравнивает его приоритет с приоритетом оператора на вершине стека.
6. Операторы с большим или равным приоритетом извлекаются из стека и добавляются в выходной массив.
7. После обработки всей строки оставшиеся операторы извлекаются из стека.

Приоритеты операторов:

| Оператор | Приоритет |
|---|---:|
| `~` - унарный минус | 4 |
| `^` | 3 |
| `*`, `/` | 2 |
| `+`, `-` | 1 |

Унарный минус внутри программы обозначается символом `~`, чтобы отличать его от бинарного вычитания.

Пример:

```text
Исходное выражение: 4*a/2
ОПН: 4 a * 2 /
```

### 5.3. Проверка корректности ОПН

После построения ОПН программа проверяет, согласованы ли операнды и операторы.

Используется счетчик глубины:

- если встречается операнд, глубина увеличивается;
- если встречается унарный оператор, должен существовать хотя бы один операнд;
- если встречается бинарный оператор, должны существовать хотя бы два операнда, после чего глубина уменьшается.

В конце корректное выражение должно дать глубину `1`, потому что все выражение должно свернуться в один результат.

### 5.4. Построение дерева выражения

Функция `build_tree` строит дерево по обратной польской записи.

Алгоритм:

1. Если встречается число или переменная, создается узел и помещается в стек.
2. Если встречается унарный оператор, из стека берется один узел и становится правым потомком оператора.
3. Если встречается бинарный оператор, из стека берутся два узла:
   - первый извлеченный становится правым потомком;
   - второй извлеченный становится левым потомком.
4. Новый узел-оператор помещается обратно в стек.
5. После обработки всех токенов в стеке остается корень дерева.

Пример для ОПН:

```text
4 a * 2 /
```

получается дерево:

```text
/
    *
        4
        a
    2
```

## 6. Алгоритм упрощения деления

Преобразование выполняется функцией `simplify_division`.

Она обходит дерево рекурсивно:

1. сначала упрощает левое поддерево;
2. затем упрощает правое поддерево;
3. после этого обрабатывает текущий узел.

Если текущий узел не является делением `/`, он остается без изменений.

Если текущий узел является делением, программа проверяет несколько случаев.

### 6.1. Деление числа на число

Если оба операнда являются числами и деление выполняется без остатка, выражение заменяется на результат.

Пример:

```text
8 / 2 -> 4
18 / 6 -> 3
```

Если деление дает остаток, выражение остается без изменений.

Пример:

```text
7 / 2 -> (7 / 2)
```

### 6.2. Деление на единицу

Если знаменатель равен `1`, деление удаляется.

Пример:

```text
x / 1 -> x
5 * x / 1 -> 5 * x
```

### 6.3. Деление на ноль

Если знаменатель равен `0`, программа не выполняет преобразование и выводит предупреждение.

Пример:

```text
x / 0 -> x / 0
```

### 6.4. Сокращение множителя в произведении

Если числитель является произведением, а знаменатель является числом, программа пытается сократить числовой множитель.

Пример:

```text
4 * a / 2 -> 2 * a
12 * x / 3 -> 4 * x
4 * a / 4 -> a
```

Для сокращения используется наибольший общий делитель.

### 6.5. Алгоритм Евклида для НОД

НОД - это наибольший общий делитель двух чисел.

В программе он вычисляется функцией `gcd_int`.

Идея алгоритма Евклида:

```text
НОД(a, b) = НОД(b, a % b)
```

Например, для чисел `12` и `8`:

```text
12 % 8 = 4
8 % 4 = 0
НОД = 4
```

## 9. Ограничения программы

Программа поддерживает:

- целые числа;
- переменные, начинающиеся с буквы;
- операции `+`, `-`, `*`, `/`, `^`;
- скобки;
- унарный минус;
- унарный плюс, который просто пропускается.

Основные ограничения:

1. Дробные числа не поддерживаются.
2. Неявное умножение не поддерживается: выражение `a(b)` считается некорректным.
3. Имена с символом `_` не поддерживаются.
4. Переменные из нескольких символов разбираются, но при выводе могут форматироваться неаккуратно, например `abc` выводится как `( abc )`.
5. Пробел перед унарным минусом после оператора может привести к ошибке разбора: `a*-2` работает, а `a* -2` отвергается.
6. Программа выполняет только ограниченные алгебраические упрощения деления и не является полной системой символьных преобразований.
