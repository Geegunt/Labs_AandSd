## 1. Цель работы

Целью лабораторной работы является разработка программы на языке C, которая:

1. хранит ориентированный взвешенный граф в виде матрицы инцидентности;
2. реализует алгоритм Флойда-Уоршелла для нахождения кратчайших путей между всеми парами вершин;
3. поддерживает восстановление кратчайшего пути через матрицу предшественников;
4. поддерживает операции добавления и удаления рёбер;
5. покрыта юнит-тестами (GTest) и включает бенчмарк производительности.

## 2. Краткие теоретические сведения

### 2.1. Матрица инцидентности

Матрица инцидентности — способ представления графа в виде матрицы размером V × E, где V — количество вершин, E — количество рёбер.

Для ориентированного взвешенного графа:
- `+w` в ячейке `[v][e]` означает, что ребро `e` выходит из вершины `v` с весом `w`;
- `-w` означает, что ребро `e` входит в вершину `v`;
- `0` означает, что вершина `v` не инцидентна ребру `e`.

### 2.2. Алгоритм Флойда-Уоршелла

Алгоритм Флойда-Уоршелла находит кратчайшие расстояния между **всеми** парами вершин в ориентированном взвешенном графе. Допускает рёбра с отрицательным весом и способен обнаруживать отрицательные циклы.

Алгоритм поддерживает матрицу расстояний `dist[V][V]` и выполняет V итераций. На каждой итерации k проверяется, можно ли улучшить путь из i в j, пройдя через вершину k:

```
if dist[i][k] + dist[k][j] < dist[i][j]:
    dist[i][j] = dist[i][k] + dist[k][j]
    next[i][j] = next[i][k]
```

Сложность: **O(V³)** по времени, **O(V²)** по памяти.

Обнаружение отрицательного цикла: если после выполнения алгоритма `dist[i][i] < 0` для какой-либо вершины i, граф содержит отрицательный цикл.

### 2.3. Восстановление пути

Матрица `next[V][V]` хранит для каждой пары (i, j) следующую вершину на кратчайшем пути из i в j. Для восстановления пути: начиная с вершины `from`, переходим к `next[from][to]`, затем к `next[next[from][to]][to]` и так далее до достижения `to`.

## 4. Описание основных структур данных

### 4.1. Граф

```c
typedef struct {
    int V;          // количество вершин
    int E;          // текущее количество рёбер
    int capacity;   // вместимость массивов рёбер
    int *src;       // откуда идёт ребро (длина E)
    int *dst;       // куда идёт ребро (длина E)
    double *weight; // вес ребра (длина E)
    double *inc;    // матрица инцидентности V × capacity
} Graph;
```

Матрица инцидентности хранится как плоский массив `inc[v * capacity + e]`. При добавлении ребра `(u, v, w)`:
- `inc[u][e] = +w` (ребро выходит из u)
- `inc[v][e] = -w` (ребро входит в v)

Массивы `src`, `dst`, `weight` дублируют информацию для быстрого доступа при построении матрицы расстояний.

### 4.2. Константы

```c
#define INF 1e18         // "бесконечность" для отсутствующих путей
#define MAX_VERTICES 1024
```

## 5. Описание функций

### 5.1. Управление графом

**`graph_create(V)`** — создаёт граф с V вершинами и начальной ёмкостью 16 рёбер.

**`graph_free(g)`** — освобождает всю память графа.

**`graph_add_edge(g, from, to, w)`** — добавляет ориентированное ребро. При заполнении массива удваивает ёмкость. Обновляет матрицу инцидентности. Возвращает 1 при успехе.

**`graph_remove_edge(g, idx)`** — удаляет ребро по индексу. Последнее ребро перемещается на место удалённого. Обновляет матрицу инцидентности. Возвращает 1 при успехе.

### 5.2. Алгоритм Флойда-Уоршелла

**`graph_floyd(g, next_out)`** — основная функция.

| Шаг | Действие |
|---|---|
| 1 | Выделить матрицы `dist[V][V]` и `next[V][V]` |
| 2 | Инициализировать: `dist[i][i] = 0`, остальное `INF`, `next = -1` |
| 3 | Заполнить начальные расстояния из рёбер графа |
| 4 | Тройной цикл по k, i, j: релаксация через вершину k |
| 5 | Вернуть `dist`, опционально — `next` |

**`graph_get_path(next, V, from, to, path, path_len)`** — восстанавливает путь из `from` в `to` по матрице `next`. Записывает вершины в массив `path`.

### 5.3. Вывод

**`graph_print_incidence(g, out)`** — печатает матрицу инцидентности в табличном виде.

**`graph_print_dist(dist, V, out)`** — печатает матрицу кратчайших расстояний.

## 11. Ограничения программы

1. Максимальное количество вершин: 1024.
2. Граф ориентированный. Для неориентированного графа нужно добавлять два ребра.
3. Алгоритм Флойда-Уоршелла обнаруживает отрицательные циклы, но не корректирует расстояния для вершин, достижимых через них.
4. Сложность по памяти: O(V²) для матрицы расстояний + O(V × E) для матрицы инцидентности.
